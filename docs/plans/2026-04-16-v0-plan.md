# Zuun v0 Implementation Plan

> **Supersedes:** `2026-04-16-v0-scaffolding.md` in the same directory. That file stays for diff — do not execute it.
>
> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Every task uses `- [ ]` checkboxes and ends with a commit.

**Goal:** Ship a local-first persistent-memory **Claude Code plugin**. One-command install (`claude --plugin-dir`). Two MCP tools (`remember`, `context_for`). A user-invoked `/zuun:reflect` slash command for session-end reflection. SessionStart context injection so new sessions open with relevant prior memory already in view. Git post-commit hook for passive capture. Markdown files on disk are human-readable, portable, and compatible with any version-control workflow users choose to layer on top.

**Product shape:** Zuun is a Claude Code plugin (not a bare MCP server). When a user runs `claude --plugin-dir /path/to/zuun` (the supported local-install path), they get the MCP server auto-registered, the slash command available in `/` autocomplete, and the SessionStart hook wired up — no three-step setup. (Marketplace publishing — `claude plugin install zuun@<marketplace>` — is a v0.1 concern once we're ready to distribute; `claude plugin install` does NOT accept a filesystem path.)

**Non-goals (v0):** Delete as MCP tool (it's a CLI command instead), export/import, MCP resources or prompts (not yet stable in Claude Code), multi-layer memory hierarchy, cross-project isolation modes, GUI, Cursor integration (schema supports it; integration is v0.1).

**Architecture:** Markdown files on disk (`~/.zuun/entries/*.md`) are the source of truth — human-readable, git-friendly, portable. A SQLite index at `~/.zuun/index.db` makes them fast to search via FTS5 (keyword) and `sqlite-vec` (semantic). Every entry carries an optional `project` field (absolute path, auto-populated from git toplevel or cwd at capture) that `context_for` / `search` / SessionStart use for project-scoped retrieval with prefix-match semantics. The index is a **derived artifact**: `zuun reindex` can always rebuild it from the filesystem, and `openDb()` throws with a reindex hint on schema-version drift. The MCP server exposes `remember` and `context_for`. The `/zuun:reflect` skill is user-invoked only (`disable-model-invocation: true`) and instructs the session's agent to distill memorable items and call `remember` for each. The SessionStart hook is passive: it runs `zuun session-start` with the session's cwd, resolves the project, fetches the N most recent project-scoped entries, and emits them as `additionalContext` — no embed dependency on this hot path.

**Tech stack:** TypeScript with `strict` and `noUncheckedIndexedAccess`, `tsx` for dev, CommonJS, Zod at I/O boundaries, `@modelcontextprotocol/sdk`, `better-sqlite3` + `sqlite-vec`, Ollama (`nomic-embed-text`, 768-dim, L2-normalized) behind an `EmbedProvider` interface, `gray-matter` for frontmatter, Vitest for tests. Claude Code plugin manifest + skill + hooks per `code.claude.com/docs/en/plugins.md`.

## What this plan delivers that the original didn't

The original `2026-04-16-v0-scaffolding.md` sketched a bare MCP server plus CLI. This plan ships a **Claude Code plugin** — a substantively better product shape. The differences fall into three tiers.

### Tier 1: correctness fixes (original plan would have failed execution)

1. **FTS5 contentless-table delete bug.** `CREATE VIRTUAL TABLE ... USING fts5(... content='')` disables regular `DELETE` — you have to use the `INSERT INTO ft(ft, rowid, col) VALUES('delete', rowid, col)` command. The old Task 5 would silently fail on every second upsert. Fixed by using regular-content FTS5.
2. **`src/lib/index.ts` is a naming landmine.** Collides with Node's barrel convention. Renamed throughout to `src/lib/store.ts`.
3. **Stale `package.json` scripts** referenced files that never existed. Fixed in Task 0.1.
4. **TDD discipline slipped** on old Tasks 6 (reindex) and 10 (MCP). Both get failing tests first here.
5. **Test concurrency hazards** (stdin monkey-patching, tsx subprocess spawning). Serialized via single-fork Vitest pool (Task 0.2).

### Tier 2: quality upgrades (same product, better built)

- **Schema versioning** (`meta` table + `SCHEMA_VERSION`) so v0.1 migrations are painless.
- **Atomic entry file writes** (`.tmp` + rename) so crash mid-write doesn't corrupt.
- **Auto-reindex hint on schema drift** — `openDb()` throws with a clear "run `zuun reindex`" message.
- **Structured `context_for` output** — id + kind + relative age + body per hit, not a text blob.
- **Hash-based dedup** on `remember` within a 10-minute window — agents retry; we shouldn't double-save.
- **Tag normalization** so `Architecture` and `architecture` collapse.
- **Append-only JSONL log** at `~/.zuun/log.jsonl` — one line per operation. Cheap, invaluable for debugging.
- **`EmbedProvider` interface** so Voyage/OpenAI/local-llama.cpp slot in cleanly later.
- **`zuun doctor`** — single health check: drift, schema, Ollama, broken refs, log tail.
- **Rich MCP tool descriptions** with explicit WHEN/WHEN-NOT/INPUT/OUTPUT sections, per the Zuhn principle *"tool descriptions are routing signals, not docstrings."*
- **FTS5 tokenizer** preserves snake_case and kebab-case identifiers (`unicode61 tokenchars '_-'`).

### Tier 3: product-shape upgrades (the reason to do this now)

- **Project-scoped memory.** Every entry carries an optional `project` field (absolute path, auto-populated from git root or cwd at capture). SessionStart filters by project so a session in `/work/acme` sees acme memory, not noise from every other repo. `context_for`, `search`, and SessionStart all share this scoping.
- **Claude Code plugin packaging.** Ships with `.claude-plugin/plugin.json` + `.mcp.json` + `skills/` + `hooks/`. One-command install for v0: `claude --plugin-dir .`. MCP auto-registers via `.mcp.json`; the slash command and hooks wire up via the manifest. Marketplace distribution (`claude plugin install <name>@<market>`) lands when we publish — v0.1.
- **`/zuun:reflect` slash command.** User-invoked (`disable-model-invocation: true`) skill that tells the session's agent to distill memorable items and call `remember` for each. Reflective capture without an external LLM call.
- **SessionStart context injection hook.** New sessions in a given cwd open with the top 3–5 relevant prior entries already in the agent's view. Uses `additionalContext` JSON output (up to 10k chars, we cap at 8k for headroom).
- **Git post-commit hook.** `zuun install-git-hook` drops a post-commit in the target repo that captures commit message + changed-file summary as an Entry. Fully passive.
- **Extended CLI surface.** `zuun forget <id>` (clean delete), `zuun edit <id>` (open in `$EDITOR`), `zuun explain <query>` (show FTS + vector + blend scores for tuning retrieval), `zuun search --kind decision --since 7d` (filtered retrieval).
- **Recency bias + kind/tag filters** in `context_for` and `search` — memory should prefer recent context; agents often want "decisions about auth," not unfiltered fuzzy match.

## Architectural Decisions (locked)

These are fixed. Do not re-litigate in task execution; if you disagree, escalate before writing code.

### Core data model

1. **Single `Entry` type.** No separate Insight/Principle/Prediction/Decision/Tension records. Every unit of memory is an `Entry` with a `kind` field. This is the #1 lesson from Zuhn's reference-integrity bugs.
2. **Markdown + YAML frontmatter on disk is the source of truth.** SQLite is a derived cache. `rm ~/.zuun/index.db && zuun reindex` must always restore full functionality.
3. **Files are canonical; SQL is not.** Never write a field to SQLite that isn't also in the markdown file. On conflict, trust the file.
4. **Schema version from day one.** `meta.schema_version = 1` is stamped on first DB init. Every future DB open checks it.

### Engineering discipline

5. **Zod validation at every external boundary** — file read, MCP input, CLI input. Do **not** run Zod on internal values that have already been validated; trust types past the boundary.
6. **Embeddings are optional and lazy.** Zuun works without Ollama running. Embeddings upgrade search quality when available. The product must never hard-fail because a local process isn't running.
7. **No daemon in v0.** The MCP stdio server lives only while Claude Code is connected. Any "passive" capture (git post-commit, SessionStart) is a short-lived CLI invocation, not a background service.
8. **TDD, no exceptions.** Every task that produces code starts with a failing test. The test is the spec.
9. **One logical change per commit.** Each task ends with a commit. Don't batch.

### Product shape

10. **Zuun ships as a Claude Code plugin.** Not a bare MCP server. For v0, installation is `claude --plugin-dir /path/to/zuun` (the documented path for local unpublished plugins). `claude plugin install` accepts marketplace names, not filesystem paths; we'll use it in v0.1 once we publish. The plugin manifest (`.claude-plugin/plugin.json`) + `.mcp.json` + `skills/` + `hooks/` are first-class deliverables, not afterthoughts. Reference: `code.claude.com/docs/en/plugins.md` and `code.claude.com/docs/en/plugins-reference`.
11. **MCP server auto-registers via the plugin manifest.** The user never runs `claude mcp add` — installing the plugin wires it up. `.mcp.json` declares the command.
12. **Agent-first tool descriptions.** MCP tool descriptions are routing signals to Claude Code, not docstrings. Invest in them. If a human can't pick the right tool from the description, neither can the agent. Enforced by Task 13's test that asserts description length ≥ 100 chars.
13. **Session-end reflection is user-invoked, not auto-triggered.** The `/zuun:reflect` skill uses `disable-model-invocation: true` — only the user types the command. Rationale: the agent deciding when to reflect mid-task would interrupt flow; the user deciding post-task gets the best signal-to-noise ratio.
14. **SessionStart injects prior context passively and cheaply.** No user action required; no embed dependency on this path. The `SessionStart` hook runs `zuun session-start`, resolves the session's cwd to a project path, and fetches the top 3 most recent entries of high-signal kind (`decision`, `pattern`, `commitment`, `reference` — observations are skipped because they're the noisy category) scoped to that project **plus all global (no-project) entries**. Recency-only ranking — no FTS/vector needed since there's no user query yet. Injects via `additionalContext` JSON, capped at 2000 chars. Misses (`no-project`, `no-entries`, `error`) are logged so the user can confirm the hook fired via `zuun doctor`. Cross-project semantic surfacing is v0.1.
15. **Extended CLI surface for user control (and why it isn't an MCP dilution).** `forget` / `edit` / `explain` are CLI commands, not MCP tools.

    **The anticipated objection:** "If zuun ships ~14 user-visible entry points across MCP + CLI + slash command + hooks, isn't the 'two MCP tools' rule self-deception?" Answer: no, because the surfaces pay different context costs.

    - **MCP tools** live in the agent's tool catalog, loaded into the system prompt on every session start. Every tool in the catalog is a decision point where the agent can pick wrong, and a token tax the user pays whether or not the tool is called. This is the budget the "two tools" rule protects.
    - **CLI commands** load lazily from the shell when the user types them. Zero context cost until invoked. Safe to expand.
    - **Slash commands** with `disable-model-invocation: true` are skill definitions Claude Code surfaces in `/` autocomplete; the agent cannot auto-invoke them. Zero context cost until the user types `/zuun:reflect`.
    - **Hooks** run outside the agent's context entirely — they output JSON the agent consumes, but the hook script itself is never loaded as a tool definition.

    **Why forget/edit specifically aren't MCP tools:** both are user-authority operations. A human deciding "this captured memory is wrong, forget it" or "refine the wording" is reviewing their own memory store — the agent does not get veto or initiative here. An agent calling `forget` to silently delete a memory it disagrees with would be a user-trust-destroying failure mode. The CLI + file-format-as-truth together make memory user-owned in a way an agent tool cannot.

    **Why explain isn't an MCP tool:** tempting because agents could use it to diagnose their own retrieval. But `explain` is a tuning surface — its output helps the *developer* iterate on blend weights and tokenizer choice. Giving the agent score breakdowns invites it to rationalize rankings instead of trust them. Ship at the CLI where the human retrieval-tuner lives; promote to MCP later if usage data shows agents benefit.

## Open Design Questions (sign-off before Task 1)

Four genuine decisions. Each has a recommended default — approve with "defaults look good" and we proceed, or redirect.

### Q1. FTS5 content mode

**Options:**
- **A (recommended):** Regular content. FTS5 table stores body text. Regular `DELETE`/`UPDATE` work. ~2× storage of body column. At 10k entries × 500 chars ≈ 5MB overhead. Negligible.
- **B:** Contentless (`content=''`). No duplicate storage. Forces `INSERT INTO ft(ft, rowid, col) VALUES('delete', ...)` syntax. Easy to get wrong.
- **C:** External-content (`content='entries'`). Normalized, trigger-synced. Right answer at 1M+ entries, overkill at personal scale.

**Recommendation:** A. Storage isn't the bottleneck at personal scale; correctness is.

### Q2. Hybrid search weighting

Locked decisions: tokenizer is `unicode61 tokenchars '_-'` (preserves snake_case and kebab-case); recency bias is in (memory *should* prefer recent context); kind and tag filters are in (agents often want "decisions about auth from last week," not fuzzy match).

**Open:** what weighting?

- **A (recommended):** `score = 0.45·fts + 0.45·vec + 0.10·recency` where recency is `exp(-age_days / 30)`. Tunable via `ZUUN_SEARCH_BLEND` env var.
- **B:** No recency, 50/50 FTS + vector (original plan).
- **C:** Heavier recency (20%+) to make memory feel snappy-and-fresh.

**Recommendation:** A. 10% recency is enough to break ties toward recent memories without drowning older high-relevance hits. `zuun explain` (Task 18) makes the weighting visible so we can tune from real usage.

### Q3. Dedup strategy on `remember`

**Options:**
- **A (recommended):** Hash the body; if an identical body was written in the last 10 minutes, return the existing ID instead of creating a second entry.
- **B:** No dedup — let `remember` always create.
- **C:** Aggressive dedup — any identical body ever seen returns the existing ID.

**Recommendation:** A. B creates duplicates from agent retries; C makes legitimate re-captures (same claim in a new context) impossible. Ten minutes catches retries without being surprising.

### Q4. SessionStart injection scope

The `SessionStart` hook runs on `startup | resume | clear | compact`. What does it inject?

- **A (recommended):** Top 3 most recent entries scoped to the session's *project* (resolved from cwd via git toplevel or path). Formatted as a terse bullet list under a "Prior relevant context from Zuun:" header. Cap at 2000 chars even though Claude Code allows 10000. No embed call — pure project-filter + recency retrieval.
- **B:** Top 8 hits — more context, more token cost before the user has done anything.
- **C:** Add semantic cross-project surfacing (with hard embed timeout) on top of A.
- **D:** Nothing — defer auto-injection to v0.1.

**Recommendation:** A. Three recent project-scoped hits is the minimum interesting dose. Retrieval is a single indexed read + in-memory filter — stays sub-millisecond. No embed dependency on the hot path means a slow/cold Ollama can't stall session startup. Cross-project semantic surfacing (C) is attractive but adds a failure surface on every session open; easy to layer in later if usage patterns ask for it.

### Task 0.3: Extend `Entry` schema with `project` field + resolver helper

**Files:**
- Edit: `src/lib/entry.ts`
- Edit: `src/lib/entry.test.ts`
- Create: `src/lib/project.ts`
- Create: `src/lib/project.test.ts`

Adds an optional `project` field to `Entry`. Populated automatically at capture time (git root or cwd) and consumed by `context_for` / SessionStart to scope retrieval by project. Without this, project affinity is accidental (pointed out by code review).

**Design:** `project` is an optional string path.

- **Inside a git repo:** auto-populated to `git rev-parse --show-toplevel` → entry is project-scoped.
- **Outside any git repo:** left `undefined` → entry is **global**, visible to every project's SessionStart / `context_for`.

This makes the filesystem context at capture time the physical affordance for scope: capture inside a repo when the insight is project-specific; capture from your home directory (or explicitly pass `project: undefined`) when it applies everywhere.

Retrieval filter (in `search.ts` and SessionStart):
- Entries with `project === undefined` are always included.
- Entries with `project` set must prefix-match in either direction (so a capture under `/work/acme` also matches a session in `/work/acme/api/src`, and vice versa).

This task only extends the existing `entry.ts` schema. The `entries` SQLite table and `store.ts` pick up the new field naturally when they're created in Tasks 6 and 7 — both are written fresh with `project TEXT` in their respective definitions (see updates below).

```typescript
import * as os from "os";
import * as path from "path";
import { execFileSync } from "child_process";
import { resolveProject } from "./project";

describe("resolveProject", () => {
  const origCwd = process.cwd();

  beforeEach(() => {
    tmp = fs.mkdtempSync(path.join(os.tmpdir(), "zuun-project-"));
  });

    process.chdir(origCwd);
    fs.rmSync(tmp, { recursive: true, force: true });
  });

  it("returns the git repo toplevel when cwd is inside a repo", () => {
    execFileSync("git", ["init", "--initial-branch=main"], { cwd: tmp });
    const subdir = path.join(tmp, "api", "src");
    fs.mkdirSync(subdir, { recursive: true });
    expect(resolveProject(subdir)).toBe(fs.realpathSync(tmp));
  });

  it("returns undefined when not inside a repo (entry is global)", () => {
    expect(resolveProject(tmp)).toBeUndefined();
  });

  it("defaults to process.cwd() when no argument passed", () => {
    process.chdir(tmp);
```

```typescript
import { execFileSync } from "child_process";

/**
 * Resolve an absolute project path for the given working directory.
 *
 * Returns the enclosing git repo's toplevel if cwd is inside a git repo.
 * Returns `undefined` if the cwd is NOT inside a git repo — treated as a
 * "global" capture in zuun's semantics (applies to every project).
 *
 * Also returns undefined if the cwd doesn't exist.
 */
export function resolveProject(cwd: string = process.cwd()): string | undefined {
  try {
    const real = fs.realpathSync(cwd);
    try {
      const root = execFileSync("git", ["rev-parse", "--show-toplevel"], {
        cwd: real,
        encoding: "utf8",
        stdio: ["ignore", "pipe", "ignore"],
      }).trim();
      return fs.realpathSync(root);
    } catch {
      // Not inside a git repo — treat as global (return undefined).
      return undefined;
    }
  } catch {
    return undefined;
  }
}
```

```typescript
// src/lib/id.ts
import * as crypto from "crypto";

export function newEntryId(body: string, at: Date = new Date()): string {
  const yy = String(at.getUTCFullYear()).slice(-2);
  const mm = String(at.getUTCMonth() + 1).padStart(2, "0");
  const dd = String(at.getUTCDate()).padStart(2, "0");
  const hash = crypto
    .createHash("sha256")
    .update(`${body}|${at.getTime()}`)
    .digest("hex")
    .slice(0, 4)
    .toUpperCase();
  return `ENT-${yy}${mm}${dd}-${hash}`;
}
```

```typescript
export function normalizeTag(raw: string): string {
  return raw.trim().toLowerCase().replace(/\s+/g, "-");
}

export function normalizeTags(raws: string[]): string[] {
  const seen = new Set<string>();
  const out: string[] = [];
  for (const raw of raws) {
    const t = normalizeTag(raw);
    if (t.length === 0 || seen.has(t)) continue;
    seen.add(t);
    out.push(t);
  }
  return out;
}
```

```typescript
import * as fs from "fs";
import * as path from "path";
import matter from "gray-matter";
import { EntrySchema, type Entry } from "./entry";
import { entriesDir } from "./paths";

function entryPath(id: string): string {
  return path.join(entriesDir(), `${id}.md`);
}

export function writeEntry(entry: Entry): void {
  EntrySchema.parse(entry);
  fs.mkdirSync(entriesDir(), { recursive: true });
  const { body, ...frontmatter } = entry;
  const file = matter.stringify(body, frontmatter);
  const final = entryPath(entry.id);
  const tmp = `${final}.${process.pid}.tmp`;
  fs.writeFileSync(tmp, file);
  fs.renameSync(tmp, final);
}

export function readEntry(id: string): Entry {
  const raw = fs.readFileSync(entryPath(id), "utf8");
  const parsed = matter(raw);
  // Do NOT .trim() — body is canonical as written.
  return EntrySchema.parse({ ...parsed.data, body: parsed.content });
}

export function listEntryIds(): string[] {
  const dir = entriesDir();
  if (!fs.existsSync(dir)) return [];
  return fs
    .readdirSync(dir)
    .filter((f) => !f.startsWith(".") && f.endsWith(".md"))
    .map((f) => f.slice(0, -3));
}
```

```typescript
// src/lib/log.ts
import * as fs from "fs";
import { logPath, storeRoot } from "./paths";

export interface LogLine {
  at: string;
  event: string;
  payload: Record<string, unknown>;
}

export function appendLog(event: string, payload: Record<string, unknown>): void {
  try {
    fs.mkdirSync(storeRoot(), { recursive: true });
    const line: LogLine = { at: new Date().toISOString(), event, payload };
    fs.appendFileSync(logPath(), JSON.stringify(line) + "\n");
  } catch {
    // Logging must never break the caller. Swallow.
  }
}

export function tailLog(n: number): LogLine[] {
  const p = logPath();
  if (!fs.existsSync(p)) return [];
  const raw = fs.readFileSync(p, "utf8");
  const lines = raw.split("\n").filter((l) => l.length > 0);
  return lines
    .slice(-n)
    .flatMap((l) => {
      try {
        return [JSON.parse(l) as LogLine];
      } catch {
        return [];
      }
    });
}
```

```typescript
// src/lib/db.ts
import Database from "better-sqlite3";
import * as sqliteVec from "sqlite-vec";
import * as fs from "fs";
import { dbPath, storeRoot } from "./paths";

export type Db = Database.Database;

export const SCHEMA_VERSION = 2;

const DDL_STATEMENTS: readonly string[] = [
  `CREATE TABLE IF NOT EXISTS meta (
     key   TEXT PRIMARY KEY,
     value TEXT NOT NULL
   )`,
  `CREATE TABLE IF NOT EXISTS entries (
     id         TEXT PRIMARY KEY,
     body       TEXT NOT NULL,
     kind       TEXT NOT NULL,
     source     TEXT NOT NULL,
     created    TEXT NOT NULL,
     stance     TEXT,
     origin     TEXT,
     project    TEXT,
     tags       TEXT NOT NULL DEFAULT '[]',
     related    TEXT NOT NULL DEFAULT '[]',
     confidence TEXT
   )`,
  `CREATE VIRTUAL TABLE IF NOT EXISTS entries_fts USING fts5(
     id UNINDEXED,
     body,
     tokenize = "unicode61 tokenchars '_-'"
   )`,
  `CREATE VIRTUAL TABLE IF NOT EXISTS entries_vec USING vec0(
     id TEXT PRIMARY KEY,
     embedding FLOAT[768]
   )`,
];

export function openDb(): Db {
  fs.mkdirSync(storeRoot(), { recursive: true });
  const db = new Database(dbPath());
  sqliteVec.load(db);
  db.pragma("journal_mode = WAL");
  db.pragma("foreign_keys = ON");

  for (const sql of DDL_STATEMENTS) db.prepare(sql).run();

  const row = db.prepare("SELECT value FROM meta WHERE key = 'schema_version'").get() as
    | { value: string }
    | undefined;

  if (!row) {
    db.prepare("INSERT INTO meta (key, value) VALUES ('schema_version', ?)").run(
      String(SCHEMA_VERSION),
    );
    return db;
  }

  if (row.value !== String(SCHEMA_VERSION)) {
    db.close();
    throw new Error(
      `zuun: index schema_version is ${row.value}, code expects ${SCHEMA_VERSION}. ` +
        `Run 'zuun reindex' to rebuild the index from your markdown files.`,
    );
  }

  return db;
}
```

```typescript
// src/lib/store.ts
import type { Db } from "./db";
import { EntrySchema, type Entry } from "./entry";

function rowToEntry(row: Record<string, unknown>): Entry {
  return EntrySchema.parse({
    ...row,
    tags: JSON.parse(row.tags as string),
    related: JSON.parse(row.related as string),
    stance: row.stance ?? undefined,
    origin: row.origin ?? undefined,
    project: row.project ?? undefined,
    confidence: row.confidence ?? undefined,
  });
}

export function upsertEntry(db: Db, entry: Entry): void {
  EntrySchema.parse(entry);
  const tx = db.transaction(() => {
    db.prepare(
      `INSERT INTO entries (id, body, kind, source, created, stance, origin, project, tags, related, confidence)
       VALUES (@id, @body, @kind, @source, @created, @stance, @origin, @project, @tags, @related, @confidence)
       ON CONFLICT(id) DO UPDATE SET
         body=excluded.body, kind=excluded.kind, source=excluded.source,
         created=excluded.created, stance=excluded.stance, origin=excluded.origin,
         project=excluded.project,
         tags=excluded.tags, related=excluded.related, confidence=excluded.confidence`,
    ).run({
      id: entry.id,
      body: entry.body,
      kind: entry.kind,
      source: entry.source,
      created: entry.created,
      stance: entry.stance ?? null,
      origin: entry.origin ?? null,
      project: entry.project ?? null,
      tags: JSON.stringify(entry.tags),
      related: JSON.stringify(entry.related),
      confidence: entry.confidence ?? null,
    });
    // Regular-content FTS5 supports plain DELETE + INSERT.
    db.prepare("DELETE FROM entries_fts WHERE id = ?").run(entry.id);
    db.prepare("INSERT INTO entries_fts (id, body) VALUES (?, ?)").run(entry.id, entry.body);
  });
  tx();
}

export function deleteEntry(db: Db, id: string): void {
  const tx = db.transaction(() => {
    db.prepare("DELETE FROM entries WHERE id = ?").run(id);
    db.prepare("DELETE FROM entries_fts WHERE id = ?").run(id);
    db.prepare("DELETE FROM entries_vec WHERE id = ?").run(id);
  });
  tx();
}

export function getEntry(db: Db, id: string): Entry | null {
  const row = db.prepare("SELECT * FROM entries WHERE id = ?").get(id) as
    | Record<string, unknown>
    | undefined;
  return row ? rowToEntry(row) : null;
}

export function listEntries(db: Db): Entry[] {
  const rows = db
    .prepare("SELECT * FROM entries ORDER BY created DESC")
    .all() as Record<string, unknown>[];
  return rows.map(rowToEntry);
}
```

```typescript
// src/lib/embed-provider.ts
import { appendLog } from "./log";

export interface EmbedProvider {
  embed(text: string): Promise<number[] | null>;
}

export const EMBED_DIM = 768;
const MAX_INPUT_CHARS = 8000;

function l2Normalize(v: number[]): number[] {
  const norm = Math.sqrt(v.reduce((s, x) => s + x * x, 0));
  return norm === 0 ? v : v.map((x) => x / norm);
}

export class OllamaProvider implements EmbedProvider {
  private readonly url: string;
  private readonly model: string;

  constructor(
    url = process.env.OLLAMA_URL ?? "http://127.0.0.1:11434",
    model = process.env.ZUUN_EMBED_MODEL ?? "nomic-embed-text",
  ) {
    this.url = url;
    this.model = model;
  }

  async embed(text: string): Promise<number[] | null> {
    const prompt = text.length > MAX_INPUT_CHARS ? text.slice(0, MAX_INPUT_CHARS) : text;
    if (prompt.length < text.length) {
      appendLog("embed.truncate", { from: text.length, to: prompt.length });
    }
    try {
      const res = await fetch(`${this.url}/api/embeddings`, {
        method: "POST",
        headers: { "content-type": "application/json" },
        body: JSON.stringify({ model: this.model, prompt }),
      });
      if (!res.ok) return null;
      const data = (await res.json()) as { embedding?: number[] };
      if (!data.embedding || data.embedding.length !== EMBED_DIM) return null;
      return l2Normalize(data.embedding);
    } catch {
      return null;
    }
  }
}

export const defaultProvider: EmbedProvider = new OllamaProvider();
```

```typescript
// src/lib/embed.ts
import type { Db } from "./db";
import { EMBED_DIM, type EmbedProvider } from "./embed-provider";

export function setEmbedding(db: Db, id: string, vec: number[]): void {
  if (vec.length !== EMBED_DIM) {
    throw new Error(`embed: expected ${EMBED_DIM}-dim, got ${vec.length}`);
  }
  const tx = db.transaction(() => {
    db.prepare("DELETE FROM entries_vec WHERE id = ?").run(id);
    db.prepare("INSERT INTO entries_vec (id, embedding) VALUES (?, ?)").run(
      id,
      new Float32Array(vec),
    );
  });
  tx();
}

export function hasEmbedding(db: Db, id: string): boolean {
  return db.prepare("SELECT 1 FROM entries_vec WHERE id = ?").get(id) !== undefined;
}

export async function embedMissing(
  db: Db,
  provider: EmbedProvider,
): Promise<{ embedded: number; skipped: number }> {
  const rows = db
    .prepare(
      "SELECT e.id, e.body FROM entries e LEFT JOIN entries_vec v ON v.id = e.id WHERE v.id IS NULL",
    )
    .all() as { id: string; body: string }[];

  let embedded = 0;
  let skipped = 0;
  for (const row of rows) {
    const vec = await provider.embed(row.body);
    if (vec === null) {
      skipped++;
      continue;
    }
    setEmbedding(db, row.id, vec);
    embedded++;
  }
  return { embedded, skipped };
}
```

```typescript
// src/lib/dedup.ts
import * as crypto from "crypto";
import type { Db } from "./db";

export function bodyHash(body: string): string {
  return crypto.createHash("sha256").update(body.trim()).digest("hex");
}

export function findRecentDuplicate(
  db: Db,
  body: string,
  now: Date,
  windowMinutes = 10,
): string | null {
  const cutoff = new Date(now.getTime() - windowMinutes * 60_000).toISOString();
  const target = bodyHash(body);
  const rows = db
    .prepare("SELECT id, body FROM entries WHERE created >= ? ORDER BY created DESC")
    .all(cutoff) as { id: string; body: string }[];
  for (const row of rows) {
    if (bodyHash(row.body) === target) return row.id;
  }
  return null;
}
```

**Note:** Hash is computed on read rather than stored as a column. Fast at personal corpus size; avoids a schema bump. Revisit if dedup ever blocks a user-facing path.

```typescript
// src/lib/search.ts
import type { Db } from "./db";
import { getEntry } from "./store";
import type { Entry, EntryKind } from "./entry";

export interface SearchOptions {
  query: string;
  queryVec?: number[];
  limit?: number;
  /** Filter: only include entries of this kind. */
  kind?: EntryKind;
  /** Filter: entry must include all listed tags (AND). */
  tags?: string[];
  /** Filter: exclude entries with `created` older than this ISO datetime. */
  since?: string;
  /** Filter: entry must be scoped under this project path (prefix match). */
  project?: string;
}

export interface SearchResult {
  id: string;
  score: number;
  entry: Entry;
  /** Exposed for `zuun explain`: per-component scores pre-blend. */
  parts: { fts: number; vec: number; recency: number };
}

interface Weights {
  fts: number;
  vec: number;
  recency: number;
}

export function parseWeights(raw: string | undefined): Weights {
  const def = { fts: 0.45, vec: 0.45, recency: 0.1 };
  if (!raw) return def;
  const parts = raw.split(",").map((s) => Number(s.trim()));
  if (parts.length !== 3 || parts.some((n) => !Number.isFinite(n))) return def;
  const sum = parts[0]! + parts[1]! + parts[2]!;
  if (sum <= 0) return def;
  return { fts: parts[0]! / sum, vec: parts[1]! / sum, recency: parts[2]! / sum };
}

/** Escape FTS5-special chars; wrap each token in quotes; OR-join. */
function sanitizeFts(q: string): string {
  return q
    .replace(/["*]/g, " ")
    .split(/\s+/)
    .filter((t) => t.length > 0)
    .map((t) => `"${t.replace(/"/g, "")}"`)
    .join(" OR ");
}

function minMax(scores: Map<string, number>): Map<string, number> {
  if (scores.size === 0) return scores;
  const values = [...scores.values()];
  const min = Math.min(...values);
  const max = Math.max(...values);
  if (max === min) {
    const uniform = new Map<string, number>();
    for (const k of scores.keys()) uniform.set(k, 1);
    return uniform;
  }
  const out = new Map<string, number>();
  for (const [k, v] of scores) out.set(k, (v - min) / (max - min));
  return out;
}

function recencyScore(createdIso: string, now: number): number {
  const ageMs = now - new Date(createdIso).getTime();
  const ageDays = Math.max(0, ageMs / 86_400_000);
  return Math.exp(-ageDays / 30);
}

function passesFilters(entry: Entry, opts: SearchOptions): boolean {
  if (opts.kind && entry.kind !== opts.kind) return false;
  if (opts.since && entry.created < opts.since) return false;
  if (opts.tags && opts.tags.length > 0) {
    const have = new Set(entry.tags);
    for (const t of opts.tags) if (!have.has(t)) return false;
  }
  if (opts.project) {
    const p = entry.project;
    // Entries with no project are global — they match every project filter.
    if (p) {
      // Project-scoped entries: prefix match in either direction, so a session in
      // /work/acme/api matches entries from /work/acme (ancestor) and vice versa.
      if (!(p === opts.project || p.startsWith(opts.project + "/") || opts.project.startsWith(p + "/"))) {
        return false;
      }
    }
  }
  return true;
}

export function search(db: Db, opts: SearchOptions): SearchResult[] {
  const limit = opts.limit ?? 10;
  if (opts.query.trim().length === 0) return [];

  const envWeights = parseWeights(process.env.ZUUN_SEARCH_BLEND);
  // When no vector is available, rebalance: drop the vector weight across FTS + recency.
  const weights: Weights = opts.queryVec
    ? envWeights
    : { fts: envWeights.fts + envWeights.vec, vec: 0, recency: envWeights.recency };

  const ftsQuery = sanitizeFts(opts.query);
  const ftsScores = new Map<string, number>();
  if (ftsQuery.length > 0) {
    try {
      const rows = db
        .prepare(
          "SELECT id, bm25(entries_fts) AS score FROM entries_fts WHERE entries_fts MATCH ? ORDER BY score LIMIT ?",
        )
        .all(ftsQuery, limit * 8) as { id: string; score: number }[];
      for (const r of rows) ftsScores.set(r.id, -r.score); // bm25 is negative; flip
    } catch {
      // FTS5 can still reject odd query shapes; fall through.
    }
  }

  const vecScores = new Map<string, number>();
  if (opts.queryVec) {
    const rows = db
      .prepare(
        "SELECT id, distance FROM entries_vec WHERE embedding MATCH ? AND k = ? ORDER BY distance",
      )
      .all(new Float32Array(opts.queryVec), limit * 8) as { id: string; distance: number }[];
    for (const r of rows) vecScores.set(r.id, 1 - r.distance);
  }

  const normFts = minMax(ftsScores);
  const normVec = minMax(vecScores);
  const candidateIds = new Set<string>([...normFts.keys(), ...normVec.keys()]);

  const now = Date.now();
  const scored: { id: string; score: number; parts: SearchResult["parts"]; entry: Entry }[] = [];
  for (const id of candidateIds) {
    const entry = getEntry(db, id);
    if (!entry || !passesFilters(entry, opts)) continue;
    const f = normFts.get(id) ?? 0;
    const v = normVec.get(id) ?? 0;
    const r = recencyScore(entry.created, now);
    const score = weights.fts * f + weights.vec * v + weights.recency * r;
    scored.push({ id, score, parts: { fts: f, vec: v, recency: r }, entry });
  }
  scored.sort((x, y) => y.score - x.score);

  return scored.slice(0, limit).map((s) => ({
    id: s.id,
    score: s.score,
    entry: s.entry,
    parts: s.parts,
  }));
}
```

```typescript
import * as fs from "fs";
```

```typescript
export function reindex(): { indexed: number; failed: string[] } {
  if (fs.existsSync(dbPath())) fs.rmSync(dbPath());
  const db = openDb();
  const failed: string[] = [];
  let indexed = 0;
  for (const id of listEntryIds()) {
    try {
      upsertEntry(db, readEntry(id));
      indexed++;
    } catch (err) {
      failed.push(`${id}: ${(err as Error).message}`);
    }
  }
  db.close();
  appendLog("reindex", { indexed, failed: failed.length });
  return { indexed, failed };
}

if (require.main === module) {
  const r = reindex();
  console.log(`indexed ${r.indexed} entries`);
  if (r.failed.length > 0) {
    console.error(`failed: ${r.failed.length}`);
    for (const line of r.failed) console.error(`  ${line}`);
    process.exit(1);
  }
}
```

```typescript
        }

async function handleRemember(args: unknown) {
  const input = RememberInput.parse(args);
  const now = new Date();
  const db = openDb();
  try {
    const existing = findRecentDuplicate(db, input.body, now);
    if (existing) {
      appendLog("remember.dedup", { id: existing });
      return {
        content: [
          { type: "text", text: `already remembered ${existing} (same body within 10 minutes)` },
        ],
      };
    }
    const id = newEntryId(input.body, now);
    const entry: Entry = {
      id,
      created: now.toISOString(),
      body: input.body,
      kind: input.kind,
      source: SOURCE,
      tags: normalizeTags(input.tags),
      related: input.related,
      stance: input.stance,
      origin: input.origin,
      project: input.project ?? resolveProject(),
    };
    writeEntry(entry);
    upsertEntry(db, entry);
    appendLog("remember", { id, kind: entry.kind, tags: entry.tags });
    // Fire-and-forget embed: search works without it; rerun via `zuun embed`.
    void defaultProvider.embed(entry.body).then((vec) => {
      if (!vec) return;
      const db2 = openDb();
      try {
        setEmbedding(db2, id, vec);
      } finally {
        db2.close();
      }
    });
    const tagLine = entry.tags.length ? ` · tags: ${entry.tags.join(", ")}` : "";
    return {
      content: [
        { type: "text", text: `saved ${id} (${entry.kind}${tagLine})\n${entry.body}` },
      ],
    };
  } finally {
    db.close();
  }
}

async function handleContextFor(args: unknown) {
  const input = ContextForInput.parse(args);
  const db = openDb();
  try {
    const project = resolveProject();
    const qVec = await defaultProvider.embed(input.task);
    const results = search(db, {
      query: input.task,
      queryVec: qVec ?? undefined,
      limit: input.limit,
      // Scope to the current project (undefined → "global only," which is the right
      // behavior when the MCP server runs outside any git repo).
      project: project ?? undefined,
    });
    appendLog("context_for", { task: input.task.slice(0, 120), project, hits: results.length });
    const text = results.length === 0 ? "no prior context" : formatResults(results);
    return { content: [{ type: "text", text }] };
  } finally {
    db.close();
  }
}

function formatResults(results: SearchResult[]): string {
  const lines = [`Found ${results.length} entries:\n`];
  const now = Date.now();
  for (const r of results) {
    const age = relativeAge(now - new Date(r.entry.created).getTime());
    const tags = r.entry.tags.length ? ` · tags: ${r.entry.tags.join(", ")}` : "";
    lines.push(`- ${r.entry.id} · ${r.entry.kind} · ${age}${tags}`);
    lines.push(`  ${r.entry.body.replace(/\n/g, " ")}`);
  }
  return lines.join("\n");
}

function relativeAge(ms: number): string {
  const d = Math.floor(ms / 86_400_000);
  if (d < 1) return "today";
  if (d < 2) return "yesterday";
  if (d < 14) return `${d}d ago`;
  if (d < 60) return `${Math.floor(d / 7)}w ago`;
  return `${Math.floor(d / 30)}mo ago`;
}

  process.exit(1);
});
```

```typescript
import { appendLog } from "./lib/log";

const VERSION = "0.0.1";

const HELP = `usage: zuun <command>

commands:
  init         create the store directory and index
  mcp          run the MCP server over stdio
  capture      read an entry body from stdin and save it
  search QRY   print top matches for QRY
  reindex      rebuild the sqlite index from markdown files
  embed        embed all entries missing vectors (requires Ollama)
  doctor       health check: entries, db, ollama, broken refs
  version      print the zuun version
  help         show this message
`;

export async function runCli(argv: string[]): Promise<number> {
  const [cmd, ...rest] = argv;
  if (!cmd) {
    process.stderr.write(HELP);
    return 1;
  }

  switch (cmd) {
    case "init":
      return cmdInit();
    case "mcp":
      await import("./mcp");
      return 0;
    case "reindex":
      return cmdReindex();
    case "embed":
      return cmdEmbed();
    case "capture": {
      const { capture } = await import("./capture");
      return capture(rest);
    }
    case "search":
      return cmdSearch(rest);
    case "doctor":
      return cmdDoctor();
    case "version":
      process.stdout.write(`zuun ${VERSION}\n`);
      return 0;
    case "help":
      process.stdout.write(HELP);
      return 0;
    default:
      process.stderr.write(`unknown command: ${cmd}\n\n${HELP}`);
      return 1;
  }
}

function cmdInit(): number {
  fs.mkdirSync(entriesDir(), { recursive: true });
  openDb().close();
  appendLog("init", { root: storeRoot() });
  process.stdout.write(`initialized zuun store at ${storeRoot()}\n`);
  return 0;
}

function cmdReindex(): number {
  const r = reindex();
  process.stdout.write(`indexed ${r.indexed}, failed ${r.failed.length}\n`);
  for (const line of r.failed) process.stderr.write(`  ${line}\n`);
  return r.failed.length === 0 ? 0 : 1;
}

async function cmdEmbed(): Promise<number> {
  const db = openDb();
  try {
    const r = await embedMissing(db, defaultProvider);
    process.stdout.write(`embedded ${r.embedded}, skipped ${r.skipped}\n`);
    return 0;
  } finally {
    db.close();
  }
}

async function cmdSearch(args: string[]): Promise<number> {
  const query = args.join(" ");
  if (!query) {
    process.stderr.write("usage: zuun search <query>\n");
    return 1;
  }
  const db = openDb();
  try {
    const qVec = await defaultProvider.embed(query);
    const results = search(db, { query, queryVec: qVec ?? undefined, limit: 10 });
    if (results.length === 0) {
      process.stdout.write("(no results)\n");
      return 0;
    }
    for (const r of results) {
      process.stdout.write(
        `${r.entry.id} · ${r.entry.kind} · ${r.entry.created}\n  ${r.entry.body.replace(/\n/g, " ")}\n\n`,
      );
    }
    return 0;
  } finally {
    db.close();
  }
}

async function cmdDoctor(): Promise<number> {
  const report = await runDoctor();
  process.stdout.write(report.text);
  return report.healthy ? 0 : 1;
}

if (require.main === module) {
  runCli(process.argv.slice(2)).then((code) => process.exit(code));
}
```

```typescript
// src/lib/doctor.ts
import { openDb, SCHEMA_VERSION } from "./db";
import { listEntryIds } from "./entry-io";
import { listEntries } from "./store";
import { defaultProvider } from "./embed-provider";
import { tailLog } from "./log";

export async function runDoctor(): Promise<{ healthy: boolean; text: string }> {
  const db = openDb();
  const lines: string[] = [];
  let healthy = true;

  try {
    const onDisk = listEntryIds();
    const inDb = listEntries(db);
    lines.push(`schema_version: ${SCHEMA_VERSION}`);
    lines.push(`entries on disk: ${onDisk.length}`);
    lines.push(`entries in db: ${inDb.length}`);
    if (onDisk.length !== inDb.length) {
      healthy = false;
      lines.push("WARN: drift between disk and db — run 'zuun reindex'");
    }

    const ids = new Set(inDb.map((e) => e.id));
    let broken = 0;
    for (const e of inDb) {
      for (const r of e.related) {
        if (!ids.has(r)) broken++;
      }
    }
    lines.push(`broken related refs: ${broken}`);

    const vec = await defaultProvider.embed("doctor-check");
    lines.push(`ollama: ${vec ? "up" : "down"}`);
    if (!vec) lines.push("  note: embeddings are optional; FTS still works");

    const tail = tailLog(5);
    if (tail.length > 0) {
      lines.push("recent log:");
      for (const l of tail) lines.push(`  ${l.at} ${l.event} ${JSON.stringify(l.payload)}`);
    }
  } finally {
    db.close();
  }

  return { healthy, text: lines.join("\n") + "\n" };
}
```

```typescript
import { upsertEntry } from "./lib/store";
import { writeEntry } from "./lib/entry-io";
import { newEntryId } from "./lib/id";
import { defaultProvider } from "./lib/embed-provider";
import { setEmbedding } from "./lib/embed";
import { EntryKind, type Entry } from "./lib/entry";
import { normalizeTags } from "./lib/tags";
import { findRecentDuplicate } from "./lib/dedup";
import { resolveProject } from "./lib/project";
import { appendLog } from "./lib/log";

interface Opts {
  kind: Entry["kind"];
  tags: string[];
}

function parseArgs(argv: string[]): Opts {
  const opts: Opts = { kind: "observation", tags: [] };
  for (let i = 0; i < argv.length; i++) {
    const a = argv[i];
    if (a === "--kind") {
      const val = argv[++i] ?? "";
      opts.kind = EntryKind.parse(val);
    } else if (a === "--tag") {
      const val = argv[++i];
      if (val) opts.tags.push(val);
    }
  }
  return opts;
}

async function readStdin(): Promise<string> {
  const chunks: Buffer[] = [];
  for await (const chunk of process.stdin) {
    chunks.push(Buffer.isBuffer(chunk) ? chunk : Buffer.from(chunk));
  }
  return Buffer.concat(chunks).toString("utf8").trim();
}

export async function capture(argv: string[]): Promise<number> {
  let opts: Opts;
  try {
    opts = parseArgs(argv);
  } catch (err) {
    process.stderr.write(`capture: ${(err as Error).message}\n`);
    return 1;
  }
  const body = await readStdin();
  if (body.length === 0) {
    process.stderr.write("capture: no body on stdin\n");
    return 1;
  }

  const now = new Date();
  const db = openDb();
  try {
    const existing = findRecentDuplicate(db, body, now);
    if (existing) {
      process.stdout.write(`${existing}\n`);
      appendLog("capture.dedup", { id: existing });
      return 0;
    }
    const id = newEntryId(body, now);
    const entry: Entry = {
      id,
      created: now.toISOString(),
      body,
      kind: opts.kind,
      source: "manual",
      tags: normalizeTags(opts.tags),
      related: [],
      project: resolveProject(),
    };
    writeEntry(entry);
    upsertEntry(db, entry);
    const vec = await defaultProvider.embed(body);
    if (vec) setEmbedding(db, id, vec);
    appendLog("capture", { id, kind: entry.kind, tags: entry.tags, project: entry.project });
    process.stdout.write(`${id}\n`);
    return 0;
  } finally {
    db.close();
  }
}
```

```typescript
// src/commands/forget.ts
import * as fs from "fs";
import * as path from "path";
import { openDb } from "../lib/db";
import { deleteEntry, getEntry } from "../lib/store";
import { entriesDir } from "../lib/paths";
import { appendLog } from "../lib/log";

export async function forget(args: string[]): Promise<number> {
  const [id] = args;
  if (!id) {
    process.stderr.write("usage: zuun forget <id>\n");
    return 1;
  }
  const db = openDb();
  try {
    const entry = getEntry(db, id);
    if (!entry) {
      process.stderr.write(`forget: no entry with id ${id}\n`);
      return 1;
    }

    // Order matters: unlink the markdown file FIRST, then delete the DB row.
    // Rationale: if we crash between steps, "file first" is self-healing via reindex
    // (file missing → DB row will be dropped on next reindex). The reverse order
    // would resurrect the entry — DB row gone, file present → next reindex re-reads
    // the file and re-inserts the row. "Forget" must not be un-forgotten.
    const file = path.join(entriesDir(), `${id}.md`);
    if (fs.existsSync(file)) fs.rmSync(file);
    deleteEntry(db, id);

    appendLog("forget", { id });
    process.stdout.write(`forgot ${id}\n`);
    return 0;
  } finally {
    db.close();
  }
}
```

```typescript
// src/commands/edit.ts
import * as fs from "fs";
import * as path from "path";
import { spawnSync } from "child_process";
import { openDb } from "../lib/db";
import { readEntry } from "../lib/entry-io";
import { upsertEntry, getEntry } from "../lib/store";
import { entriesDir } from "../lib/paths";
import { appendLog } from "../lib/log";

export async function edit(args: string[]): Promise<number> {
  const [id] = args;
  if (!id) {
    process.stderr.write("usage: zuun edit <id>\n");
    return 1;
  }
  const db = openDb();
  try {
    if (!getEntry(db, id)) {
      process.stderr.write(`edit: no entry with id ${id}\n`);
      return 1;
    }
    const file = path.join(entriesDir(), `${id}.md`);
    if (!fs.existsSync(file)) {
      process.stderr.write(`edit: file missing on disk: ${file}\n`);
      return 1;
    }

    const editor = process.env.EDITOR ?? "vi";
    const result = spawnSync(editor, [file], { stdio: "inherit" });
    if (result.status !== 0) {
      process.stderr.write(`edit: editor exited with status ${result.status}\n`);
      return 1;
    }

    try {
      const parsed = readEntry(id);
      upsertEntry(db, parsed);
      appendLog("edit", { id });
      process.stdout.write(`updated ${id}\n`);
      return 0;
    } catch (err) {
      process.stderr.write(
        `edit: file fails schema after edit — db row left unchanged\n  ${(err as Error).message}\n`,
      );
      return 1;
    }
  } finally {
    db.close();
  }
}
```

```typescript
// src/lib/git-store.ts
import * as fs from "fs";
import * as path from "path";
import { execFileSync } from "child_process";
import { storeRoot } from "./paths";
import { appendLog } from "./log";

const GITIGNORE_CONTENTS = `# Derived artifacts — do not version
index.db
index.db-journal
index.db-wal
index.db-shm
log.jsonl
*.tmp
`;

function gitSync(args: string[], opts: { mustSucceed?: boolean } = {}): { ok: boolean; stdout: string } {
  try {
    const stdout = execFileSync("git", args, {
      cwd: storeRoot(),
      encoding: "utf8",
      stdio: ["ignore", "pipe", "pipe"],
    });
    return { ok: true, stdout };
  } catch (err) {
    if (opts.mustSucceed) throw err;
    return { ok: false, stdout: "" };
  }
}

export function isGitStore(): boolean {
  return fs.existsSync(path.join(storeRoot(), ".git"));
}

export function initGitStore(): void {
  fs.mkdirSync(storeRoot(), { recursive: true });
  if (!isGitStore()) {
    gitSync(["init", "--initial-branch=main"], { mustSucceed: true });
    gitSync(["config", "user.email", "zuun@localhost"], { mustSucceed: true });
    gitSync(["config", "user.name", "Zuun"], { mustSucceed: true });
    appendLog("git.init", { root: storeRoot() });
  }
  const ignorePath = path.join(storeRoot(), ".gitignore");
  if (!fs.existsSync(ignorePath)) {
    fs.writeFileSync(ignorePath, GITIGNORE_CONTENTS);
    gitSync(["add", ".gitignore"]);
    gitSync(["commit", "-m", "chore: initial .gitignore"]);
  }
}

export function commitPath(absFile: string, message: string): boolean {
  if (!isGitStore()) return false;
  const root = storeRoot();
  const rel = path.relative(root, absFile);
  if (rel.startsWith("..")) return false;
  gitSync(["add", "--", rel]);
  const { ok } = gitSync(["commit", "-m", message, "--", rel]);
  if (ok) appendLog("git.commit", { file: rel, message });
  return ok;
}
```

```markdown
name: reflect
description: Reflect on what was decided, learned, or built in this session and save the high-signal items to Zuun so future sessions have them. User-invoked only.

# Reflect on this session

You are being invoked at the end (or a natural breakpoint) of a Claude Code session. Your job is to look back at what happened and preserve the 2–5 highest-signal items — decisions made, observations worth keeping, patterns learned, commitments to future-self — by calling the Zuun `remember` tool for each.

## What to save

Save items that:
- A new session tomorrow would have to re-derive from scratch.
- Would cause the user to re-explain themselves if lost.
- Are *claims*, not *events*. "We decided to use SQLite over Postgres because the store is single-user" — not "added SQLite."

## What NOT to save

- Step-by-step task notes — those belong in code comments or commit messages.
- Bugs that were already fixed — the fix lives in the diff.
- Speculation or things you haven't verified.
- Content already captured earlier in this session — check your own memory of past `remember` calls first.

## How to call `remember`

For each item, call the `remember` tool with:

- `body` — one self-contained claim. Under 300 chars. Not a paragraph.
- `kind` — pick the best fit: `decision`, `observation`, `pattern`, `commitment`, or `reference`.
- `tags` — 2–4 lowercase tags scoping the claim.
- `stance` (optional) — a one-line directional claim if the body is evaluative.
- `origin` (optional) — file path, PR number, or session marker if the memory ties to a specific artifact.
- `related` (optional) — entry IDs this one builds on (only if you retrieved them earlier this session via `context_for`).

## Cadence

Aim for **2–5 entries total**. Five is the ceiling, not the target. If you can't find 2, save fewer — one great entry beats four mediocre ones. If the session produced nothing worth preserving (a debugging round that ended in "it was a typo"), say so and skip.

## Dedup

The `remember` tool deduplicates same-body entries within a 10-minute window. `already remembered ENT-...` is fine — move on.

## Confirm and stop

After your last `remember` call, briefly list the ids you saved (or report "nothing worth preserving"), then stop. Do not call any other tools. Do not ask follow-up questions.
```

### Task 22: SessionStart context injection hook

**Files:**
- Create: `hooks/hooks.json`
- Create: `hooks/session-start.sh`
- Create: `src/hook-scripts/session-start.ts`
- Create: `src/hook-scripts/session-start.test.ts`
- Edit: `src/cli.ts` — add `session-start` subcommand

**Design (v0):** The hook resolves the session's `cwd` to a `project` path (via `resolveProject`), then retrieves the N most recent entries of **high-signal kinds** (`decision`, `pattern`, `commitment`, `reference`) scoped to that project plus all global (no-project) entries, and emits them as `additionalContext` JSON for Claude Code to inject.

**Why the kind filter?** The adversarial review correctly pointed out that recency-only retrieval would surface yesterday's debugging crumbs — noise from the `observation` kind — into today's planning session. Decisions, patterns, and commitments are semantically stable across sessions; observations are the noisy category. Skipping `observation` at the SessionStart surface keeps the eager-load signal-dense without dropping the feature.

**What we deliberately do NOT do at v0:**
- **No embed call on this path.** `SessionStart` fires on every session; a cold or slow Ollama would stall startup. Recency-based retrieval within a project is strong enough signal for v0. Cross-project semantic matching ("the pattern you learned in Acme is relevant in Beta") is v0.1 material — easy to add behind a hard timeout later.
- **No FTS ranking.** Without a user query there's nothing to rank against. "Recent in this project" is the right default.
- **No fallback to cross-project entries.** Keeps noise low. If a user wants cross-project recall, they ask `context_for` explicitly.

**Project scoping:** Uses prefix-match via the `project` field (added in Task 0.3). A session in `/work/acme/api/src` matches entries scoped to `/work/acme` (ancestor) and `/work/acme/api/src/feature` (descendant). Entries with **no** `project` (captured outside any git repo) are treated as **global** and included in every session's SessionStart — they're the natural bucket for insights that apply everywhere ("use Zod at MCP boundaries").

**Logging:** Every hook fire appends one line to `log.jsonl`. `session_start.inject` when context was emitted; `session_start.miss` when nothing matched (with a reason: `no-project`, `no-entries`, or `error`). This lets `zuun doctor` tail the log and prove the hook fired even when the agent sees no injection.

**Fires on:** `startup | resume | clear | compact`. On `compact` it's redundant but cheap; on `startup` and `resume` it's load-bearing.

```typescript
import { openDb } from "../lib/db";
import { listEntries } from "../lib/store";
import { resolveProject } from "../lib/project";
import { appendLog } from "../lib/log";
import type { Entry, EntryKind } from "../lib/entry";

const MAX_CHARS = 2000;
const TOP_K = 3;

/**
 * SessionStart only surfaces high-signal kinds. Observations get skipped because
 * they're the noisy category — yesterday's debugging crumbs would drown out
 * today's planning session if we included them. Decisions, patterns, commitments,
 * and references are semantically stable across sessions and worth eager display.
 */
const HIGH_SIGNAL_KINDS: readonly EntryKind[] = ["decision", "pattern", "commitment", "reference"];

export interface HookInput {
  cwd: string;
}

function entryInProject(entry: Entry, project: string): boolean {
  const p = entry.project;
  // Entries with no project are global — included in every session's SessionStart.
  if (!p) return true;
  return p === project || p.startsWith(project + "/") || project.startsWith(p + "/");
}

function relAge(ms: number): string {
  const d = Math.floor(ms / 86_400_000);
  if (d < 1) return "today";
  if (d < 2) return "yesterday";
  if (d < 14) return `${d}d ago`;
  if (d < 60) return `${Math.floor(d / 7)}w ago`;
  return `${Math.floor(d / 30)}mo ago`;
}

export async function runSessionStart(input: HookInput): Promise<void> {
  try {
    const project = resolveProject(input.cwd);
    if (!project) {
      appendLog("session_start.miss", { cwd: input.cwd, reason: "no-project" });
      return;
    }

    const db = openDb();
    try {
      // listEntries returns sorted by created DESC. Filter by project + kind, then take TOP_K.
      const scoped = listEntries(db)
        .filter((e) => HIGH_SIGNAL_KINDS.includes(e.kind))
        .filter((e) => entryInProject(e, project))
        .slice(0, TOP_K);

      if (scoped.length === 0) {
        appendLog("session_start.miss", { project, reason: "no-entries" });
        return;
      }

      const now = Date.now();
      const lines: string[] = ["Prior relevant context from Zuun:\n\n"];
      for (const e of scoped) {
        const body = e.body.replace(/\n/g, " ");
        const age = relAge(now - new Date(e.created).getTime());
        lines.push(`- ${e.id} (${e.kind}, ${age}): ${body}\n`);
      }
      let text = lines.join("");
      if (text.length > MAX_CHARS) text = text.slice(0, MAX_CHARS - 1) + "…";

      const payload = {
        hookSpecificOutput: {
          hookEventName: "SessionStart",
          additionalContext: text,
        },
      };
      process.stdout.write(JSON.stringify(payload));
      appendLog("session_start.inject", { project, hits: scoped.length, chars: text.length });
    } finally {
      db.close();
    }
  } catch (err) {
    appendLog("session_start.miss", { cwd: input.cwd, reason: "error", err: (err as Error).message });
    process.stderr.write(`zuun session-start hook: ${(err as Error).message}\n`);
  }
}
```

```typescript
// src/commands/capture-commit.ts
import { execFileSync } from "child_process";
import { openDb } from "../lib/db";
import { upsertEntry } from "../lib/store";
import { writeEntry } from "../lib/entry-io";
import { newEntryId } from "../lib/id";
import { normalizeTags } from "../lib/tags";
import { findRecentDuplicate } from "../lib/dedup";
import { appendLog } from "../lib/log";
import type { Entry, EntryKind } from "../lib/entry";

function kindFromMessage(msg: string): EntryKind {
  const prefix = msg.trim().split(/[:(\s]/, 1)[0]?.toLowerCase() ?? "";
  if (prefix === "feat") return "pattern";
  if (prefix === "fix") return "observation";
  if (prefix === "chore" || prefix === "refactor") return "decision";
  if (prefix === "docs" || prefix === "test") return "reference";
  return "observation";
}

function gitSafe(args: string[]): string | null {
  try {
    return execFileSync("git", args, {
      encoding: "utf8",
      stdio: ["ignore", "pipe", "ignore"],
    }).trim();
  } catch {
    return null;
  }
}

export async function captureCommit(_args: string[]): Promise<number> {
  const sha = gitSafe(["rev-parse", "HEAD"]);
  if (!sha) return 0; // not in a repo
  const msg = gitSafe(["log", "-1", "--pretty=%B"]) ?? "";
  if (msg.trim().length === 0) return 0;
  const branch = gitSafe(["rev-parse", "--abbrev-ref", "HEAD"]) ?? "unknown";
  // --root ensures first commits show their files too (default diff-tree treats root as empty).
  const files = gitSafe(["diff-tree", "--no-commit-id", "--name-only", "-r", "--root", sha]) ?? "";
  const repoRoot = gitSafe(["rev-parse", "--show-toplevel"]);

  const body = `${msg.trim()}\n\nFiles changed:\n${files}`;
  const now = new Date();

  const db = openDb();
  try {
    const existing = findRecentDuplicate(db, body, now);
    if (existing) {
      appendLog("capture_commit.dedup", { sha, id: existing });
      return 0;
    }
    const id = newEntryId(body, now);
    const entry: Entry = {
      id,
      created: now.toISOString(),
      body,
      kind: kindFromMessage(msg),
      source: "git",
      tags: normalizeTags([branch]),
      related: [],
      origin: sha,
      project: repoRoot ?? undefined,
    };
    writeEntry(entry);
    upsertEntry(db, entry);
    appendLog("capture_commit", { id, sha, kind: entry.kind, project: entry.project });
    process.stdout.write(`${id}\n`);
    return 0;
  } finally {
    db.close();
  }
}
```

- [ ] **Step 5: Write `install-git-hook` implementation**

```typescript
// src/commands/install-git-hook.ts
import * as fs from "fs";
import * as path from "path";
import { execFileSync } from "child_process";

const ZUUN_MARKER = "# zuun:post-commit";
const SHEBANG = "#!/usr/bin/env sh\n";

/**
 * Absolute path to the zuun.js we want the installed hook to invoke.
 * Git hooks run outside Claude Code, so they cannot rely on PATH or CLAUDE_PLUGIN_ROOT
 * at fire time. We resolve the plugin's install location NOW and embed it literally
 * in the hook content.
 *
 * Priority:
 *   1. CLAUDE_PLUGIN_ROOT (set when running from an installed plugin) → <root>/bin/zuun.js
 *   2. process.argv[1] — the shim we were invoked via
 */
function resolveZuunBinPath(): string {
  const pluginRoot = process.env.CLAUDE_PLUGIN_ROOT;
  if (pluginRoot) return path.resolve(pluginRoot, "bin", "zuun.js");
  const argv1 = process.argv[1];
  if (argv1) return path.resolve(argv1);
  throw new Error("install-git-hook: cannot resolve absolute path to zuun.js");
}

/** Resolve .git/hooks/post-commit honoring core.hooksPath and worktrees. */
function resolveHookPath(): string | null {
  try {
    const rel = execFileSync("git", ["rev-parse", "--git-path", "hooks/post-commit"], {
      encoding: "utf8",
      stdio: ["ignore", "pipe", "ignore"],
    }).trim();
    return path.resolve(process.cwd(), rel);
  } catch {
    return null;
  }
}

export async function installGitHook(_args: string[]): Promise<number> {
  const hookPath = resolveHookPath();
  if (!hookPath) {
    process.stderr.write("install-git-hook: not inside a git repository\n");
    return 1;
  }

  let zuunBin: string;
  try {
    zuunBin = resolveZuunBinPath();
  } catch (err) {
    process.stderr.write(`install-git-hook: ${(err as Error).message}\n`);
    return 1;
  }

  const zuunLine = `${ZUUN_MARKER}\nnode ${JSON.stringify(zuunBin)} capture-commit >/dev/null 2>&1 || true\n`;

  fs.mkdirSync(path.dirname(hookPath), { recursive: true });

  let content: string;
  if (fs.existsSync(hookPath)) {
    content = fs.readFileSync(hookPath, "utf8");
    if (content.includes(ZUUN_MARKER)) {
      process.stdout.write("post-commit hook already installed\n");
      return 0;
    }
    if (!content.endsWith("\n")) content += "\n";
    content += "\n" + zuunLine;
  } else {
    content = SHEBANG + zuunLine;
  }

  fs.writeFileSync(hookPath, content);
  fs.chmodSync(hookPath, 0o755);
  process.stdout.write(`installed post-commit hook at ${hookPath}\n  invoking: ${zuunBin}\n`);
  return 0;
}
```

## Roadmap (explicitly deferred past v0)

Good ideas we are **not** doing now. Each has a reason.

- **`forget` and `edit` as MCP tools.** Available as CLI commands (Task 18). MCP tool surface stays at two per the attention-budget principle. Add only if usage data shows agents asking users to delete/edit.
- **MCP resources / prompts.** Not yet stable in Claude Code. Revisit when the spec settles.
- **`export` / `import` commands.** `tar -czf ~/zuun.tgz ~/.zuun` works today. `git push` to a private remote is arguably better than an export format.
- **Per-project store isolation.** Unclear benefit at personal scale. `ZUUN_HOME` override is the escape hatch if anyone needs it.
- **Multi-layer memory hierarchy (short-term / long-term / reasoning traces).** Right direction; v0.2 material. The `kind` field gives us the semantic hook for layering later.
- **Cursor integration.** Schema already supports `source: cursor`. Adding a Cursor-specific capture path is a v0.1 task (different hooks/commands, same MCP).
- **Web UI / browser extension.** Out of scope per README.
- **Batch git commits (one per session vs one per capture).** Current per-capture commit is fine at human capture rates; revisit if burst-captures become common.
- **`zuun sync` / cross-device sync tooling.** Users can `git push` today. Dedicated command only if the UX needs polish.
- **Tag autocomplete / ontology.** Intentional non-goal — "no ontology, no hierarchy" per the schema's own docstring.
